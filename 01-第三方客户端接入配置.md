# 第三方客户端接入 Codesome 配置指南

本文汇总常见第三方客户端的 Codesome 配置方法，不包含客户端安装步骤。

## 1. Claudian 配置

### 1.1 打开设置页

进入 Claudian 设置页，找到截图所示的位置。

如果你在[ Claude Code配置指南](https://zvgmnl1sw58.feishu.cn/wiki/IPomwd31niucKwkVIVucP63an1g)中使用的是**环境变量方式**，请关闭 **“加载用户 Claude 设置”**，然后继续看 **1.2**。

如果你使用的是**配置文件方式**，例如 \`ccswitch\` 或 \`settings.json\`，请保持 **“加载用户 Claude 设置”** 开启，然后继续看 **1.3**。

![](<images/第三方客户端接入 Codesome 配置指南-image-001.png>)

### 1.2 使用环境变量配置

在设置页继续下滑，找到环境变量配置区域。

![](<images/第三方客户端接入 Codesome 配置指南-image-002.png>)

按下图位置新增环境变量，填入以下内容：

```powershell
ANTHROPIC_BASE_URL="https://cc.codesome.ai"
ANTHROPIC_AUTH_TOKEN="你的apikey"
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1

```

填写完成后，点击右下角 \`+\` 保存。

![](<images/第三方客户端接入 Codesome 配置指南-image-003.png>)

保存时，名称填写为：

```powershell
codesome
```

然后点击右下角保存。

![](<images/第三方客户端接入 Codesome 配置指南-image-004.png>)

配置完成后即可使用。

### 1.3 使用 `settings.json` 配置

如果你使用的是`settings.json`、`ccswitch` 或其他本地配置文件方式，请保持 **“加载用户 Claude 设置”** 开启。

按以下顺序检查：

1\. 确认`ccswitch`的代理开关已关闭。

2\. 确认你在 Claude Code CLI 中已经可以正常使用。

3\. 回到 Claudian，保持 **“加载用户 Claude 设置”** 开启，不再额外填写环境变量。

4\. 保存后直接测试。

注意：

* 不要在这里混用环境变量和配置文件两套方案。

* 不要把普通 `Max` 或普通 `max2.2` 分组当成可外接分组。2026 年 7 月 15 日客服实测只有名称完整标注为 **`max2.2 可外接`** 的分组支持第三方客户端；普通 `PRO` 不支持 Fable。Fable 已于 2026 年 7 月 19 日后不可用，不要按旧教程继续配置它。最终以当天后台的分组说明和模型列表为准。

### 1.4 Windows 用户填写 Claude Code 路径

如果你是 Windows 用户，并且 Claudian 需要你手动填写 Claude Code 路径，请填写 **真实可执行文件路径**，不要填写包装层文件。

先在 PowerShell 中执行：

```powershell
$root = npm root -g
dir "$root\@anthropic-ai\claude-code"
dir "$root\@anthropic-ai\claude-code\bin"
```

如果输出中存在：

```plain&#x20;text
claude.exe
```

则 Claudian 中应填写：

```plain&#x20;text
C:\Users\你的用户名\AppData\Roaming\npm\node_modules\@anthropic-ai\claude-code\bin\claude.exe
```

不要填写下面这些路径：

```plain&#x20;text
claude.cmd
claude.ps1
cli-wrapper.cjs
```

填写完成后，建议先在 PowerShell 中验证：

```powershell
& "C:\Users\你的用户名\AppData\Roaming\npm\node_modules\@anthropic-ai\claude-code\bin\claude.exe" --version
```

如果能正常输出版本号，再回到 Claudian 中继续使用。

### 1.5 验证是否配置成功

完成配置后，直接在 Claudian 中新建一个会话，发送一条最简单的测试消息，例如：

```plain&#x20;text
请回复 OK
```

能正常收到回复，即表示配置成功。

### 1.6 常见问题处理

#### 1.6.1 使用环境变量方式后仍然不生效

按以下顺序检查：

1\. 确认 **“加载用户 Claude 设置”** 已关闭。

2. 确认 Claudian 中填写的是：

   * `ANTHROPIC_BASE_URL="https://cc.codesome.ai"`

   * `ANTHROPIC_AUTH_TOKEN="你的apikey"`

   * `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`

3. 删除 Claudian 中旧的、重复的环境变量，只保留当前实际使用的配置。

4) 保存后重启 Obsidian，再重新测试。

#### 1.6.2 使用`settings.json`方式后仍然不生效

按以下顺序检查：

1\. 确认 **“加载用户 Claude 设置”** 已开启。

2. 确认 `ccswitch` 没有继续接管当前配置。

3. 先在 Claude Code CLI 中测试是否可用。

4) 如果 CLI 能用，再回到 Claudian 测试。

#### 1.6.3 Windows 下报\`Error: spawn EFTYPE\`

按以下顺序处理：

1. 检查 Claudian 中填写的 Claude Code 路径。

2. 如果填的是 `claude.cmd`、`claude.ps1` 或 `cli-wrapper.cjs`，改成真正的 `bin\claude.exe`。

3) 保存设置。

4) 完全退出 Obsidian。

5. 重新打开 Obsidian 后再测试。

#### 1.6.4 改完配置还是旧效果

按以下顺序处理：

1. 保存当前设置。

2. 完全退出 Obsidian。

3) 重新打开 Obsidian。

4) 再重新测试。

不要只切换页面或只关闭设置窗口。

### 1.7 推荐操作顺序

首次配置时，建议直接按下面顺序操作：

1. 先确认 Claude Code CLI 已经能正常使用。

2. 进入 Claudian 设置页。

3) 根据你的实际方案二选一：

* 环境变量方式：关闭 **“加载用户 Claude 设置”**

* 配置文件方式：开启 **“加载用户 Claude 设置”**

4. Windows 用户额外确认 Claude Code 路径填写的是 `bin\claude.exe`。

5. 保存设置。

6) 重启 Obsidian。

7) 新建会话，发送一条测试消息验证。

## 5. Trae 配置

Trae 支持通过自定义模型接入 Codesome。以下分别介绍 V3 和二合一两种接入方式。

### 5.1 V3 接入配置

V3 接入要按协议填写不同的 Base URL，API Key 为 `sk-...` 格式：Anthropic 客户端通常填写 `https://cc.codesome.ai`，由客户端追加 `/v1/messages`；OpenAI / Codex 客户端通常填写 `https://cc.codesome.ai/v1`，由客户端追加 `/responses` 或 `/chat/completions`。不要在 V3 中使用旧的 `/openai` 路径。

#### 5.1.1 配置 Claude 模型

打开 Trae 的模型配置页面，按下图填写：

* Provider 选择 `Anthropic`（或 Custom）

* Base URL：`https://cc.codesome.ai`

* API Key：填写你的 `sk-...` 格式 API Key

* 模型：以参考图片为准。

![](<images/第三方客户端接入 Codesome 配置指南-image-2.png>)

![](<images/第三方客户端接入 Codesome 配置指南-image-1.png>)

#### 5.1.2 配置 GPT 模型

如需使用 GPT 模型，按下图填写：

* Provider 选择 `OpenAI`（或 Custom）

* Base URL：`https://cc.codesome.ai/v1`

* API Key：填写你的 Codex 分组 API Key

* 模型 ID：以参考图片为准。

![](<images/第三方客户端接入 Codesome 配置指南-image.png>)

![](<images/第三方客户端接入 Codesome 配置指南-image-10.png>)

### 5.4 注意事项

* V3 和二合一使用不同的 Base URL 和 API Key 格式，不要混用。

* V3 的 Claude 和 GPT 使用不同协议路径：Anthropic 客户端通常用 `https://cc.codesome.ai`，OpenAI / Codex 客户端通常用 `https://cc.codesome.ai/v1`；API Key 分组也可能不同。

* 二合一的 Claude 模型使用 `/api` 路径，GPT 模型使用 `/openai` 路径，注意区分。
